# Super Desktop Assistant 连接器评级与准入规范

本文定义连接器能力等级（L1/L2/L3）的量化标准、验收流程和降级规则，用于统一评估 MCP、Skill、桌面客户端与成熟 CLI 接入质量。

## 1. 评级目标

- 让路由策略“可解释”，不是黑盒优先级
- 让连接器接入有一致门槛，避免“能用但不可靠”
- 让产品 UI 能明确提示风险与适用场景

## 2. 评级模型

### 2.1 L1（GUI 自动化 / Best Effort）

适配方式：

- GUI 自动化点击或弱结构化桥接

最低要求：

- 能完成基础路径操作
- 有失败检测与人工接管入口
- 有最小证据（截图/动作日志）

限制：

- 不可作为默认自动执行路径
- 仅允许低风险任务或显式用户确认

### 2.2 L2（本地进程桥接 / 基础可控）

适配方式：

- 本地进程桥接（含 PTY 或 stdout/stderr 结构化转译）

最低要求：

- 可稳定启动与终止
- 错误可观测（标准错误码映射）
- 审批与策略门控可接入
- 支持至少一种恢复策略（重试或切沙箱）

### 2.3 L3（官方 API/CLI / 生产优先）

适配方式：

- 官方 API、官方 CLI、稳定协议桥接

最低要求：

- 高稳定性与版本兼容策略
- 结构化事件完备
- 审计覆盖完整
- 幂等/重试语义明确

路由策略：

- Auto 模式默认优先 L3

## 3. 量化评分维度（100 分）

- 稳定性（30）：成功率、超时率、崩溃率
- 可观测性（20）：错误码、事件完整度、日志可追踪
- 可治理性（20）：策略门控、审批接入、审计覆盖
- 可恢复性（15）：重试、局部恢复、降级
- 体验保真（15）：交互一致性、延迟、会话连续性

等级建议：

- `L3`：>= 85 且稳定性 >= 25
- `L2`：>= 65
- `L1`：< 65 或仅 GUI 自动化

## 4. 准入验收流程

1. 连接器自检（健康检查 + 权限检查）
2. 回归用例执行（核心路径、异常路径、审批路径）
3. 评分与等级确定
4. 安全评审（workspace 边界、敏感数据处理）
5. 灰度发布（小流量）
6. 观察窗口后正式启用

## 5. 必测用例清单

### 5.1 功能用例

- 成功执行核心任务
- 非法参数处理
- 连接器不可用时反馈

### 5.2 安全与策略用例

- workspace 越界阻断
- 高风险命令触发审批
- 审批拒绝后正确中断

### 5.3 恢复用例

- 节点失败后原地重试
- 切换执行器重试
- 沙箱重试

### 5.4 PTY 专项（成熟 CLI 必测）

- 方向键/回车交互正常
- 终端 resize 正常
- 会话断连重连正常

## 6. 运行期降级规则

- 连续失败率超阈值 -> L3 降级 L2
- 审计事件缺失超阈值 -> 禁止自动路由，仅手动调用
- 严重安全事件 -> 立即禁用连接器

## 7. UI 展示规范

在 `Integrations` 和 `Mission` 显示：

- 当前等级（L1/L2/L3）
- 最近 24h 成功率
- 是否支持自动路由
- 是否支持审批与完整审计
- 推荐场景与风险提示

## 8. 复评机制

- 周期复评：每 2 周自动评分
- 版本变更复评：连接器大版本升级后强制复评
- 事故复评：发生 P1/P2 事故后 24h 内复评

## 9. 不达标处置

- 不达标连接器禁止参与 Auto 路由
- 需补齐缺失项后重新提交验收
- 保留人工使用入口，但必须显式风险提示
