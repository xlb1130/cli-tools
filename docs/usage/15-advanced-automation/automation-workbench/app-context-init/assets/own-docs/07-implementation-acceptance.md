# 实现验收

## 目标

确保“开发完成”不只是代码写完，而是业务目标、质量目标和运维目标共同达标。

## 验收维度

- 需求覆盖：目标能力是否完整实现。
- 行为正确性：关键路径是否符合预期。
- 兼容性：老版本、老数据、老配置是否可共存。
- 可运维性：日志、指标、告警是否齐备。
- 可回滚性：出问题时是否有退路。

## 证据要求

- 关键测试结果。
- 关键链路日志或截图。
- 数据对账结果。
- 配置和发布检查记录。

## 验收结论

- `通过`
- `附条件通过`
- `不通过`

## 架构师关注点

- 新增能力是否破坏了系统整体一致性。
- 是否引入了隐性复杂度而未显式暴露。
