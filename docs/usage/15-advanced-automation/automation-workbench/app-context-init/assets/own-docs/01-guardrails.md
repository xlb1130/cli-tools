# 护栏规则

## 目标

为自动化执行建立不可逾越的安全边界，确保系统在具备自主分析和执行能力时，仍然符合最小风险、最小影响面和可追责原则。

## 核心原则

- 默认只读：未明确授权前，优先执行查询、分析、比对，不直接做写操作。
- 先证据后动作：没有日志、数据、配置或运行态证据时，不进入修复和发布动作。
- 最小影响面：能针对单实例、单服务、单租户处理，就不要扩大到全局。
- 可回滚：任何会改变线上状态的动作，都必须先说明回滚路径。
- 可审计：关键决策、关键命令、关键结论必须可追踪。

## 禁止项

- 未经确认直接修改生产配置、数据库结构或批量数据。
- 在证据不足时宣称根因已确认。
- 在未完成兼容性评估前推动高风险发布。
- 将密钥、口令、Token、连接串写入仓库或报告正文。

## 升级条件

- 涉及生产写操作。
- 涉及跨系统连锁影响。
- 涉及财务、合规、权限或安全风险。
- 观察到与既有经验相冲突的异常模式。

## 人工确认点

- 发布、回滚、限流、熔断、开关切换。
- 批量补数据、删数据、重放任务。
- 修改告警阈值或绕过安全策略。

## 输出要求

- 明确当前动作属于只读、低风险写、还是高风险变更。
- 明确前置证据、预期收益、潜在副作用和回滚方案。
