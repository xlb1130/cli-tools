# Switch 开关使用规范

## 目标

让开关成为可控的交付和止血工具，而不是长期积累的技术债入口。

## 开关分类

- 功能开关：控制能力是否开放。
- 灰度开关：控制开放范围和节奏。
- 紧急开关：用于快速止血和降级。

## 使用原则

- 每个开关都必须有责任人、作用域和回收时间。
- 开关默认应该可观测，至少能看到命中率和生效范围。
- 不能把开关当作永久分支逻辑。

## 设计要求

- 名称能表达业务含义和影响范围。
- 生效对象明确到环境、租户、用户群或流量比例。
- 关闭后系统行为必须可预测。

## 回收机制

- 发布稳定后及时删除无用开关。
- 紧急开关必须进入复盘，避免“长期紧急态”。
