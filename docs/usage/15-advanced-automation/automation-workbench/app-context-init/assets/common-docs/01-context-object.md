# 上下文对象

## 目标

统一所有 skill 的输入、状态和输出承载方式，减少上下文在多阶段流转时的损耗和歧义。

## 推荐字段

- `task_id`：任务唯一标识。
- `goal`：任务目标。
- `scope`：影响范围和边界。
- `environment`：环境信息。
- `signals`：症状、证据、指标、异常摘要。
- `constraints`：权限、时间、发布窗口、合规约束。
- `decision_log`：关键决策和理由。
- `next_actions`：后续动作。

## 来源优先级

- 用户输入优先于历史缓存。
- 运行时证据优先于人工猜测。
- 明确记录推断字段，避免伪装成事实。

## 更新规则

- 每次阶段性结论都应更新上下文对象。
- 修改必须保留前后差异，至少保留关键决策变更。
